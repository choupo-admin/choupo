## THEMES (read these first — they drive batch fixes)

**T1. The test harness lies (HIGHEST — meta-defect).** Multiple cases crash (`exit 2`) or fail their golden master yet `runTests` reports PASS, because the NaN/inf guard only scans the result block and an errored/missing block falls through to "smoke pass." This is the forbidden `binary && PASS` pattern. **One fix retroactively surfaces ~8 hidden-broken cases.** Affected: rankine01_water, esterification2sector/REACTION/reactor, .../SEPARATION/flash, FERMENTATION/Fermentor, DRYING/CY, plus the two ChemicalPlantTutorial golden-master failures. **Fix = make runTests fail on non-zero exit / missing result block.**

**T2. Spray-dryer unit op is physically broken.** Both `ChemicalPlantTutorial/DRYING/SD` and `ChemicalPlantTutorial/DRYING/SD` produce sub-zero-Celsius products (-53 °C), η_th = 1.63 (>100%), zero or backwards water evaporation, and contradictory non-closing mass balances (135% and 90%). Root cause is twofold: the `DryingAir` stream defaults to `vf=0` (liquid N2), AND the sprayDryer energy/water-partition + solid-phase mass accounting is wrong. v2 "works" in some siblings only because its air carries `state superheatedVapour`. **This is a unit-op bug, not a case bug** — fix the op, then re-record.

**T3. `vf` defaults to 0 (liquid) for gas feeds — recurring trap.** A stream that omits `vf`/`state` silently becomes liquid. Bites: SD spray-dryer (fatally), gasSolidSplitter01, sprayDryer01_sugar, utility02 (liquid water at 499 °C). **Cross-cutting fix: parser should warn/refuse when a stream fed to a vapour-requiring unit (sprayDryer, gas-solid, heater above Tsat) defaults to vf=0.**

**T4. Spurious `txy`/VLE blocks emitted for non-VLE units.** turbine01_air (mole fractions up to 170!), solidDryer01_sugar (nonvolatile sucrose), steam02_water_transport (liquid transport next to ideal-gas Z). The emitter ships garbage VLE envelopes that the GUI would plot. **Fix: suppress txy when any component is nonvolatile/no-psat, or when bubble/dew solve yields y∉[0,1], or for single-phase rotating equipment.**

**T5. README prose contradicts the case's own numbers (most common single theme).** ~12 cases quote expected values that are simply wrong vs output: compare_eos_co2 (Z), optim02 (bounds direction), gibbs01 (60 vs 67%), gibbs02 (claims near-complete conversion, gets 1.8%), turbine01_air (30 vs 10 kW), compressor02 (9.1 vs 9.36), membrane05 (copy-paste from membrane01), membrane09, combined01 (T/P), compare_visc_liquid_water (trend backwards), evaporator02 (25 vs 55 wt%), rankine01 README. **These are cheap, high-value fixes — recompute prose against CSV/JSON.**

**T6. `tutorials/props/old/` is shipped, listed, and regression-run.** ~11 deprecated cases (several excellent, some with stale numbers, one a duplicate of `steady/optimisation/fitNRTL01_ethanol_water`) surface in `listCases` with an "old" name that sows distrust. **Fix: archive the folder out of harness scope, or promote the good ones and resolve the duplicate.**

**T7. Provenance field reads `"undeclared"` even when prose cites the primary source.** steam01_if97, pitzer01/02, yang2020_Tg_pvc/polystyrene, ternary01, ratio cases. The glass-box provenance machinery students are taught to trust is empty exactly where a citation exists. **Fix: wire the declared citation into the structured `source` field.**

**T8. Illustrative/synthetic data unlabeled.** ternary01 (artificial compA/B/C), ryield01 (illustrative yields), pitzer01 (identical fit==anchored columns) — students can't tell engineered/uncalibrated data from real.
